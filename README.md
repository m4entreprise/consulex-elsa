# ConsuLex × ELSA — Site du concours d’éloquence

## À propos (résumé)
Ce projet est le site web événementiel du **concours d’éloquence ConsuLex × ELSA**.

Il a été conçu pour :
- présenter l’édition en cours (thème, date, lieu, jury, partenaires) ;
- centraliser les **inscriptions** des **candidats** et des **spectateurs** via deux formulaires dédiés ;
- aider l’équipe organisatrice grâce à un **espace administrateur** permettant de suivre les inscriptions et de gérer certaines options (ex. nourriture).

- **Propriétaire et créateur** : M4 Entreprise
- **Bénéficiaire principal** : ConsuLex ULiège
- **Nom de domaine prévu** : `consulex-elsa.be`

## Le concours, en quelques mots
Le concours d’éloquence est un événement durant lequel des candidats présentent une déclamation (un texte préparé) devant un public et un jury.

Le site sert de **porte d’entrée officielle** : il permet d’informer, de convaincre, et surtout de **gérer les inscriptions** avec des limites de places (capacité de salle / quota de candidatures).

---

## Objectifs du site
Le site a été imaginé comme un outil à la fois :
- **informatif** : tout comprendre sur l’édition en cours en quelques minutes ;
- **événementiel** : un rendu premium, immersif, élégant et lisible ;
- **pratique** : s’inscrire rapidement, sans étapes inutiles ;
- **organisateur-friendly** : suivre les inscriptions, récupérer les documents, éviter les erreurs.

Le site ne gère **pas** de paiement en ligne.

---

## À qui s’adresse le site
Le site s’adresse principalement à :
- **Les spectateurs** : personnes souhaitant assister à l’événement.
- **Les candidats** : personnes souhaitant participer en présentant une déclamation.
- **Les organisateurs** : équipe ConsuLex ULiège / ELSA, qui gère les inscriptions, les quotas et certains contenus.
- **Les partenaires** : entreprises et institutions mises en avant.

---

## Ce que les visiteurs peuvent faire
Sans créer de compte, un visiteur peut :
- découvrir l’édition (thème, informations pratiques) ;
- consulter les partenaires ;
- accéder aux documents (règlement, politique de confidentialité) ;
- s’inscrire en tant que **spectateur** ;
- s’inscrire en tant que **candidat**.

---

## Parcours “Spectateur” (inscription)
### Objectif
Permettre à une personne de réserver sa place (et éventuellement celles d’accompagnants), en respectant la capacité maximale.

### Principes importants
- **Capacité limitée** : une jauge bloque automatiquement les inscriptions lorsque la salle est complète.
- **Inscription rapide** : l’objectif est un parcours simple, réalisable en 1 à 2 minutes.

### Informations demandées
Lors d’une inscription spectateur, le formulaire demande typiquement :
- identité (nom, prénom) ;
- coordonnées (email, téléphone) ;
- nombre d’accompagnants (limité) ;
- un choix de nourriture (si applicable) ;
- une validation obligatoire des conditions (RGPD).

### En cas de quota atteint
Lorsque la jauge est atteinte, le site :
- indique clairement que les inscriptions sont clôturées ;
- empêche toute nouvelle soumission.

---

## Parcours “Candidat” (inscription)
### Objectif
Permettre à un candidat de soumettre sa participation, en respectant un **quota de candidatures**.

### Principes importants
- **Quota limité** : clôture automatique lorsque le nombre maximum de candidatures est atteint.
- **Dépôt de documents** : le candidat transmet les éléments nécessaires au dossier.

### Informations demandées
Lors d’une inscription candidat, le formulaire demande typiquement :
- identité (nom, prénom) ;
- coordonnées (email, téléphone) ;
- faculté / établissement ;
- dépôt d’un texte au format PDF (déclamation 5 à 8 minutes) ;
- dépôt d’une attestation d'inscription à l'ULg ;
- une validation obligatoire des conditions (RGPD et règlement).

### En cas de quota atteint
Lorsque le quota candidat est atteint, le site :
- affiche un message de fin d’inscription ;
- empêche l’envoi de nouvelles candidatures.

## Contenus et pages du site
Le site est structuré autour de pages simples, avec un accès direct aux appels à l’action.

### Page d’accueil
La page d’accueil met en avant :
- le **thème** de l’édition ;
- une introduction ;
- deux boutons d’action très visibles : **Inscription Candidat** et **Inscription Spectateur** ;
- les informations pratiques (date, lieu, accès, services sur place) ;
- une présentation du concours ;
- la mise en avant du jury ;
- une rétrospective des éditions précédentes (ex. vidéo “aftermovie”) ;
- un accès rapide au compte Instagram officiel ;
- un aperçu des partenaires.

### Page partenaires
Une page dédiée liste l’ensemble des partenaires, avec pour chacun :
- un logo ;
- le nom ;
- une courte description ;
- éventuellement un lien.

### Pages légales
Le site donne accès à des documents officiels (au format PDF) :
- politique de confidentialité ;
- règlement du concours.

## Espace administrateur (pour les organisateurs)
L’espace administrateur est destiné à l’équipe organisatrice. Son rôle : **gagner du temps** et **réduire les erreurs**.

### Ce que l’équipe organisatrice peut faire
- consulter un tableau récapitulatif des inscriptions **spectateurs** :
  - nom, prénom ;
  - téléphone ;
  - nombre total de places (inscrits + accompagnants) ;
  - choix de nourriture.
- consulter un tableau récapitulatif des inscriptions **candidats** :
  - nom, prénom ;
  - téléphone ;
  - faculté ;
  - liens de téléchargement des documents transmis (texte PDF, photo).

### Gestion de la nourriture (si utilisée)
Le site propose une interface permettant de :
- ajouter, modifier ou supprimer des options de nourriture ;
- éventuellement activer/désactiver une option.

### Suivi et pilotage
L’interface prévoit des aides de suivi, telles que :
- des compteurs (places restantes / candidatures restantes) ;
- des fonctions de recherche et de tri ;
- l’export des listes (ex. CSV) pour faciliter l’organisation.

## Données personnelles (RGPD) et sécurité — version non technique
Le site collecte uniquement les informations nécessaires à l’organisation de l’événement.

### Ce qui est fait pour respecter les bonnes pratiques
- **Transparence** : l’utilisateur est informé de la finalité (inscription et organisation).
- **Consentement** : une case d’acceptation (RGPD + règlement) est obligatoire.
- **Protection des échanges** : les données transitent via une connexion sécurisée (HTTPS).
- **Prévention des abus** : des protections existent contre les envois automatiques (spam) et les envois multiples.

### Conservation des données
Les données ne sont pas destinées à être conservées indéfiniment. Une durée de conservation raisonnable est prévue après l’événement (puis suppression/archivage selon décision des organisateurs).

## Partenaires et valorisation
Les partenaires sont valorisés via :
- une présence sur la page d’accueil (aperçu) ;
- une page dédiée ;
- des logos affichés de façon claire et qualitative.

L’objectif est de donner une visibilité crédible, cohérente et “premium”, au niveau attendu d’un site événementiel.

## Questions fréquentes (FAQ)
### Le site est-il payant ?
Non. Le site n’implique aucun paiement en ligne.

### Puis-je m’inscrire si la salle est complète ?
Non. Le site bloque automatiquement les nouvelles inscriptions quand la jauge est atteinte.

### Puis-je m’inscrire avec des accompagnants ?
Oui, dans la limite prévue par le formulaire.

### Que se passe-t-il si le quota de candidatures est atteint ?
Les inscriptions candidat se clôturent automatiquement et le site l’indique clairement.
